## 3 신규 메카닉 #1 — *옆자리 노드* (업 축 진입)

### 한 줄

**화면 한 자리에 *낯선 흙 색조* 의 자국 한 무리가 박혀 있다 — 플레이어가 그 옆에 잠깐 멈추면, 자기 자국빛이 옆 자국의 결을 살짝 받아간다.**

### 동작

- *낯선 흙 색조 자국 무리* (visual-bible §7 의 *다른 hex 패밀리*) 가 화면 한 자리에 미리 박힌 채 등장. 동행 자국 1 점 (concept §2) 의 *5–8 점 묶음* 변형. 게임 안에서 *오래 옆에 있던 사람의 자리* 의 시각화.
- 플레이어가 그 묶음의 *반경 1 보 안* 에서 *6–10 초 입력 없음* 시 — 자기 자국빛 (visual-bible §6 `#C8995A`) 면적이 *0.5–1% 만큼 일시 증가* 후 페이드 (총 약 4 초). 자국 hex 자체는 변하지 않음 — 플레이어의 *자국빛 결* 만 옆자리의 *받음* 으로 살짝 살아남.
- 옆자리에서 다시 입력하면 (=일어나서 걸으면) 자기 자국 진하기는 그대로, 단 *받은 결* 이 다음 1–2 자국의 자국빛 면적에 미세히 누적 (≤ 0.5% 가산 / 자국).
- 본 메카닉 = ③ 노드의 *옆 변주*. ④ 노드 동행 결정 분기와 별도.

### 시각

- 옆자리 묶음 자체는 *동행 자국 hex 룰* (fail-modes §F5) 그대로 — *플레이어 자국보다 살짝 옅게*.
- *받음* 의 시각은 *자국빛의 한 박자 호흡* — 종이 위 한 점이 잠깐 따뜻해졌다가 식는 느낌. 종이 외 영역 변화 0.
- UI / 화살표 / 텍스트 / 사운드 강조 모두 부재. 발견은 시각만으로.

### 매니페스토 진동

- *연대* — 옆에 잠깐 앉음의 시각. *합쳐짐 아닌 받음*.
- *축복* — 멈춤이 *받음* 으로 결을 받는다. 멈춤 ≠ 처벌 (fail-modes §F4) 의 1 차 회복 보강.
- *진심* — 옆자리에서 짧게 머무는 게 본 메카닉의 결. *오래 머물수록* 받는 게 아니라 *6–10 초 한 박자* 가 표적.

### 트립와이어 (designer self-check)

- *유도* 위험: 옆자리 묶음이 너무 진하면 *반드시 가야 할 자리* 처럼 보임. 안전핀 = §F5 의 동행 hex 룰 직접 적용 + 묶음에 *유도 화살표/광휘 부재*.
- *의무화* 위험: 옆자리에 머물지 않으면 진척 차단? **불가**. 코어 루프는 옆자리 0 회로도 회전. 옆자리는 *감산 없는 가산* — 받지 않아도 손해 0.
- *권력 비극* 트랩 (bible §6, character-sheets §인규 트립 #1): 옆자리에 *너무 자주* 머물면 그게 *답이 되는 자리* 가 되지 않는가? **세션 단위 가산 상한** = 한 세션 옆자리 가산 ≤ 2.5% (= 5 회 × 0.5% 상한). 그 이상은 *받음* 이 *습관* 으로 굳음.

### implementer 인계 (walk-trace-spec §M3 보강 자리)

- 새 상태 1 종: `companion_cluster: { center: {x,y}, hex_family: 'other', point_count: 5..8 }[]`. 프리렌더 데이터셋의 *cluster* 1 종으로 시드.
- 새 상태 1 종: `received_glow_carry: number (0..0.025)`. 세션 단위 누적, 다음 자국 1–2 회 동안 자국빛 면적 가산.
- 검출 로직: `tick` 마다 `player_pos` 와 cluster center 간 거리 + `last_step_at` 만으로 6–10 초 멈춤 판정. 새 입력 0.
- v0+ 가 박히면 walk-trace-spec §M1 (입력) 변경 0, §M2 (자국 hex) 변경 0, §M3 (매핑) 에만 *옆자리 자국빛 일시 가산* 1 줄 추가.

## 4 신규 메카닉 #2 — *함께 걸은 자리 노드* (가족 축 진입)

### 한 줄

**플레이어가 걷는 동선과 *같은 방향* 으로 시간차로 박혀 있던 다른 색조의 자국이 만나면, 두 자국이 *나란히* (합쳐지지 않고) 박힌다 — 자기 자국 진하기는 그대로, 단 자국 *옆에 한 점* 이 더 박힘.**

### 동작

- 동행 자국 풀 (concept §2 의 비동기 동행 자국) 중 *플레이어 1 차 동선의 방향과 ± 30° 안에서 같은 방향* 으로 시간차로 박힌 자국이 있으면, 플레이어가 그 자리를 지날 때 — 자기 자국이 박힘과 동시에 그 옆에 *동행 색조 한 점이 한 박자 늦게* 페이드인. 두 점은 *겹치지 않음* (visual-bible §6 의 *겹쳐짐 아닌 합쳐짐 금지* 직접 적용).
- 두 자국 간 거리: 종이 너비의 1/16. 종이 시점 (무릎-허리, visual-bible §5) 에서 *나란히* 식별 가능한 폭.
- 자기 자국 진하기 / hex / 자국빛 모두 변동 0. 단 *옆에 한 점이 더 있음* 의 시각이 추가. 메카닉이 박는 것은 *진하기 합산* 이 아니라 *나란함의 그림*.

### 시각

- 동행 색조 점은 *자기 자국이 박힌 0.5 초 후* 페이드인 — 한 박자 늦은 시각. *나란히 함께 걸은* 결.
- 같은 자리에 4 점 이상 누적되면 두 색조가 *나란한 한 줄* 로 보이기 시작. 5 점 이상이 *함께 걸은 자리의 산* 의 작은 단면 — terrain-v0 §통과의 산 의 시각으로 자연 연결.

### 매니페스토 진동

- *연대* — 1 차. *합쳐짐 아닌 겹쳐짐* 의 한 변주: *겹쳐짐 아닌 나란함*.
- *시간이 곧 나* — 동행 색조 자국이 *시간차* (한 박자 늦음) 로 등장. 같은 자리를 지난 두 시간이 한 종이 위에 박힘.
- *진심* — *방향* 일치 (± 30° 안) 가 본 메카닉의 발화 조건. 무작위 동행 자국과 다른 자리.

### 트립와이어 (designer self-check)

- *합쳐짐 트랩* (writer §유경 §트립 #1, bible §2.3): 두 색조가 시각으로 *한 색조* 로 보이면 매니페스토 위반. 안전핀 = 두 점 간 거리 ≥ 종이 1/16 + hex 패밀리 분리 (visual-bible §7).
- *희생 결 트랩* (writer §유경 §트립 #1, *아이의 자국이 자기 자국을 덮음*): 동행 색조가 자기 자국보다 진하지 않음 — fail-modes §F5 직접 인용.
- *동행 의무화* 위험: 함께 걸은 자리가 없으면 게임 진행 차단? **불가**. 본 메카닉은 ③·④ 노드 회전과 별도의 *부속 시각*. 0 회 발화로도 코어 루프 회전.

### implementer 인계 (walk-trace-spec §M3 보강 자리)

- 새 검출 1 종: `recent_facing_history: facing_deg[N=8]` 로 *최근 8 자국의 방향 평균*. cluster facing 과의 차 ≤ 30° 면 발화.
- 새 시각 1 종: 자기 자국 spawn 후 0.5 초 지연 + 같은 cluster 의 1 점을 거리 1/16 옆에 페이드인. shader 변경 0 (기존 trace render 1 점 추가일 뿐).
- v0+ 의 walk-trace-spec 보강은 §M3 매핑 룰의 *함께 걸은 자리 분기* 1 줄 + `companion_traces` 의 facing 메타데이터 추가뿐.

## 5 게임 1차 무대 — *통과의 산* (terrain-v0 §5 직접 인용)

본 게임 안의 *큰 산* = *통과의 산* 의 한 단면.

- 화면 시작 시 *낯선 흙 색조* 의 자국이 화면 면적 ≤ 8% (visual-bible §2 룰 — *빈 종이 ≥ 80%*) 안에서 *빽빽이 흩어진* 상태. 옆자리 cluster (§3) 의 5–8 점 묶음이 그중 1–2 곳, 함께 걸을 수 있는 동선의 자국 (§4) 이 그중 5–8 점.
- 플레이어 1 차 동선은 통과의 산의 한 자락을 가로지르는 짧은 길 — *길* 이라 부르지 않음 (terrain-v0 §3 — *길 = 사람이 만드는 것*). 종이 위 좌표만 박힘.
- 본 게임은 *어디로 가야 하는가* 를 박지 않는다 — 통과의 산의 한 단면 위에서 *어디에 잠깐 앉을지* / *어느 동선과 나란히 걸을지* / *그냥 지날지* 의 분기만 박힘.

## 8 페일 모드 보강 — F6 / F7 (fail-modes-v0 §확장 자리)

### F6. 옆자리가 *답이 되는 자리* — *권력 비극* 트랩 게임 단위

**증상**: 플레이어가 옆자리에서 받은 자국빛이 *너무 좋아서* 한 cluster 만 반복 방문. 다른 자리·다른 분기 모두 옅음. *옆자리* = *해야 하는 일* 로 변질.

**1 차 방어막**: §3 §트립 #3 의 세션 단위 가산 상한 (≤ 2.5%) + 한 cluster 당 받음 발화 ≤ 1 회 / 세션. 옆자리 6–10 초 한 박자가 *짧음* 의 결로 박힘.

**관측 지표**: 한 cluster 의 재방문 비율. 임계 ≤ 30%.

### F7. 함께 걸은 자리가 *합쳐짐* 으로 보임 — *합쳐짐 트랩* 시각 단위

**증상**: 두 색조 자국 거리가 종이 1/16 보다 가까워서 cold reader 가 *한 색조* 로 인식. *겹쳐짐 아닌 합쳐짐* 의 매니페스토 위반.

**1 차 방어막**: §4 의 거리 임계 ≥ 종이 1/16 엄수 + hex 패밀리 분리 (visual-bible §7 직접 인용) + 동행 색조의 *옅음 룰* (fail-modes §F5).

**관측 지표**: cold reader 5 인 시뮬에서 *두 색조* 즉답 비율 ≥ 80%.
