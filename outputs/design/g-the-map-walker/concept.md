# g-the-map-walker — Concept v0

> 본 v0 은 *세계 #2 여정 자체가 지도* 의 첫 vertical slice 컨셉. *30 초에 매니페스토를 한 번 진동시키는 한 게임* 으로 박는다.
> 본문은 1 페이지를 의도. 깊은 메카닉 분기·밸런스 표는 v1 에서 분해.

---

## 1 한 줄

**걸으면 종이에 흙이 박히고, 어느 자국 옆에 *내가 박지 않은 흙* 이 보일 때 게임이 시작된다.**

이 한 줄을 30 초 안에 보여주지 못하는 빌드는 본 게임이 아니다 (charter §트립와이어).

---

## 2 30 초 빌드 — 분-단위 시퀀스

세계 사전 지식 0 인 플레이어가 빈 화면에서 시작해 *첫 의문 1 개* 가 머릿속에 박히는 자리까지.

| 초 | 화면 | 입력 | 시각 피드백 (visual-bible §2 hex 적용) |
|----|------|------|--------------------------------------|
| 0–3 | 빈 새벽 (`#F5EFE3` 배경 ≥ 80%). Walker 가 화면 1/8 비율, 3/4 후방 시점, 무릎 직전 직립. | 없음 — 잠깐의 정적. | 종이 섬유 결만. *움직임 검증 §5*: 한 발이 막 디딜 참 자세. |
| 3–8 | 플레이어가 첫 입력 (방향 키 / 한 번 탭). | 1 회 — 한 걸음. | 발 디딘 자리에 *먼지길* `#5C4A36` 자국 1 점 + 진심으로 디딘 자리에 *자국빛* `#C8995A` 가 1–3% 면적으로 들어찬다. |
| 8–18 | 플레이어가 자기 호흡 속도로 두·세·네 걸음. | 2–4 회. | 자국 그라디언트가 형성된다. 빨리 누르면 흙이 옅고, *천천히 의도된 입력* 일수록 진해진다 (= *진심* 의 시각). |
| 18–25 | 화면 가장자리에 *낯선 흙 색조* 의 옅은 자국 1 점이 페이드인. Walker 의 흙 hex 와 명백히 다른 패밀리 (visual-bible §7 식별자 룰). | 없음 — 시선 유도만 (카메라 미세 pan). | 새 자국은 *살짝 옅게*. 시간 차로 *동행자가 거기 *있었다* 는 인상*. *합쳐짐 아니라 겹쳐짐*. |
| 25–30 | 플레이어 머릿속에 *"이게 누구지?"* 1 개 질문 발생. Walker 는 그 자국 *방향으로 한 발 / 자기 방향으로 한 발* 의 갈림길에 선다. | (선택은 30 초 직후) | 두 자국 사이의 빈 종이가 60% 이상. *축복* 의 시각 정의가 살아 있다. |

**30 초 종료 시점에 박혀 있는 것**: ① 시각 1 장 (백지 + 자국 + 자국빛) ② 룰 1 개 (걸으면 흙이 박힌다) ③ 의문 1 개 (다른 자국이 누구의 것인가). critic 체크리스트 v0 의 ①·③·⑥ 통과 표적이 정확히 이 셋.

## 3 코어 루프 — 4 노드 다이어그램 (charter §task #2)

```
        ┌──────────────────────────────────────────────┐
        │                                              ▼
   ┌─────────┐    ┌──────────┐    ┌──────────────┐    ┌──────────────┐
   │ ① 걷기  │ ─▶ │ ② 자국  │ ─▶ │ ③ 겹친 자국 │ ─▶ │ ④ 동행 결정 │
   │ (입력)  │    │ 박힘     │    │ 발견 (외부) │    │ (선택)       │
   └─────────┘    └──────────┘    └──────────────┘    └──────────────┘
        ▲              │                                      │
        │              │  (자기 자국 누적 → 정체성)            │
        │              ▼                                      │
        │         [내 지도]    ◀──────────────────────────────┘
        │           ▲
        └───────────┴───── ④ 가 ① 의 *방향* 을 살짝 바꾼다 (강요 아님)
```

**노드별 매니페스토 진동**:
- ① 걷기 = *움직임* (1 차). 입력이 곧 자연법칙.
- ② 자국 박힘 = *시간이 곧 나* + *진심*. 누른 호흡이 흙의 진하기.
- ③ 겹친 자국 발견 = *연대* (1 차 시각화). 합쳐짐 아닌 겹쳐짐.
- ④ 동행 결정 = *방향*. 따라가기·자기 길·잠깐 멈추기 — *세 가지 모두 정답*. 처벌 없는 분기.

**루프 회전 시간**: 한 사이클 ≈ 60–90 초. 30 초 빌드 = 첫 반 사이클 (① → ② → ③ 까지). 한 세션 (5 분) 안에 ④ 가 두세 번 돌아간다.

## 4 메카닉 (1 차 박음 — v0 범위)

| 메카닉 | 1 줄 | 트립와이어 (없으면 코어 루프 안 돈다 / v2 미룬다) |
|--------|------|---------------------------------------------|
| **걸음 입력** | 한 입력 = 한 발자국. 연속 입력 사이의 간격이 흙 진하기에 직접 매핑. | **없으면 코어 루프 안 돈다.** *움직임* 의 1 차 시각이 입력 그 자체. |
| **자국 hex 그라디언트** | 진심 (간격) → hex 채도, 방향 (연속성) → hex 면적. visual-bible §6 의 두 그라디언트와 1:1. | **없으면 코어 루프 안 돈다.** *시간이 곧 나* + *진심* 시각화의 핵심. |
| **비동기 동행 자국** | 다른 플레이어 / 봇 (cy-001 v0 은 *프리렌더 자국* 으로 시작 — 봇 / 매칭 인프라는 v1) 의 자국이 시간차로 페이드인. | **없으면 코어 루프 안 돈다.** ③ 노드. *연대* 시각화의 1 차. |
| **동행 결정** | 따라가기 / 자기 길 / 멈추기 — 셋 다 옳음. 처벌 없는 분기. | **없으면 코어 루프 안 돈다.** ④ 노드. 강요 트립와이어. |
| **쉼의 자리** | 멈추면 흙이 옅어지지만 *없어지지는 않는다*. 잠깐의 정적이 자국빛 광휘로 살짝 갱신. | **v2 로 미루어도 코어는 돈다 — 다만 voice-keeper *축복* 결 보강에 직결.** v0 에서 박는 것을 강력 권고. |
| **자기 지도 보기** | 게임 내 별도 메뉴 없음. *세션 종료 시* 자기가 박은 자국 전체가 한 번에 종이 위에 그려져 보인다. | **v2 로 미룰 수 있다 — 단 자기 객체화 메타포 (loremaster v0.2 의 *손바닥에 비친 지도*) 가 박히면 곧바로 v0 에 흡수.** |

기타 (퀘스트, NPC, 인벤토리, HUD, 점수, 레벨업) — 본 v0 에서 **모두 부재**. *움직임* 외의 미사여구는 본 vertical slice 의 결을 흐린다 (charter §트립와이어 — *lore 가 두꺼워서 게임이 읽기 자료로 변질*).

## 5 카메라 / UI — visual-bible §5 직접 인용

- 인물 눈높이 또는 무릎-허리 높이.
- **부감 금지 / 미니맵 금지 / 좌상단 HUD 금지**. 본 세계의 첫 트릭 (*자기 지도는 본인만 본다*) 의 직접 위반.
- UI 는 *없음* 에 가깝다. 입력 안내는 첫 8 초 안에 *흐릿한 발자국 가이드* 로만 (가이드 자국은 자국빛 0% — 시각으로 *플레이어 자국이 아님* 이 즉답되도록).
- 사운드: 흙이 디뎌지는 *숨소리 길이의 한 박자*. 음악 없음 (v0). 리듬 게임 화 트립.

## 6 매니페스토 매핑 — 본 게임이 어떻게 매니페스토를 진동시키는가

| 키워드 | 진동 자리 | v0 에서의 위치 |
|--------|----------|---------------|
| **움직임** (1차) | ① 걷기 = 입력 = 자연법칙 | ✅ 박힘 (메카닉 자체) |
| **시간이 곧 나** (2차) | 세션 호흡 속도 → 흙 진하기 → 자기 지도 모양 | ✅ 박힘 (자국 hex 그라디언트) |
| **진심** (2차) | 빠르게 누른 입력 vs 의도된 호흡 입력의 시각 차이 | ✅ 박힘 (간격 → 채도) |
| **연대** | ③ 겹친 자국 — 다른 흙 색조의 비대칭 등장 | ✅ 박힘 (비동기 동행 자국) |
| **방향** | ④ 동행 결정 — 처벌 없는 분기 | ✅ 박힘 (분기 메카닉) |
| **축복** | *쉼의 자리* + 빈 종이 ≥ 60% + 자국빛 1–3% | ⚠️ 약함 — *쉼의 자리* 메카닉 박혀야 voice-keeper 트립 회복. v0 에서 박는 것 권고. |
| **자립** | *자기 지도 보기* (세션 종료 시 한 번) | ⚠️ 약함 — loremaster v0.2 *손바닥에 비친 지도* 도착 시 흡수. |

**voice-keeper 합산 예측**: *축복* (현 0.3) + *자립* (현 0.7) 두 키워드를 본 컨셉이 어떻게 처리하는가에 따라 합산 ±1.0 변동. 본 v0 의 *쉼의 자리* 메카닉이 박히면 *축복* 0.3 → 0.7 가능 (bible v0.1-score.md §5 와 일치).

## 7 vertical slice 범위 (cy-001 또는 cy-002 첫 표적 후보)

본 v0 에서 다루는 표적:
- **30 초 빌드** (§2) — 단일 씬, 단일 Walker, 프리렌더 비동기 동행 자국 1 개. critic r2 가중치 평가의 1 차 입력.
- **5 분 세션** — ① → ② → ③ → ④ 가 두세 번 돌아간 뒤 자기 지도 한 번 보고 종료.
- **vertical slice 범위 밖**: 봇 매칭 인프라, 멀티플레이어 동기화, 저장/계정, 다국어, 레벨 디자인 다수, 스토어 / 챕터.

**implementer 입력 (engine choice 의 cost/value 표 §designer 항목)**:
- *기술 요구* — 2D 종이 셰이더 (섬유 결 + 흙 그라디언트 + 자국빛 광휘 1–3%), 비동기 자국 데이터 로드, 카메라 무릎-허리 시점, 소수 입력 (1–2 키 / 1 탭). 무겁지 않다.
- *드리프트 위험* — 모델 미모 드리프트는 본 게임에 부재 (인물 익명 실루엣 1 명). 셰이더 드리프트 (자국빛 과잉) 가 1 차 위험 — *art-director §9 트립* 와 동일.
- *우선 후보 결*: 웹 (Three.js + WebGL2) 또는 Godot. Unity 는 본 v0 의 시각 결 (종이/흙) 에 비례 비용 과대.
